
что такое ветка в GIT:
Ветка (своими словами): отклонение от основной линии разработки (MASTER / MAIN) для решения новых задач.
В  MASTER / MAIN  хранится стабильная версия. 
Мы создаём ветки для решения наших задач (1 ветка - 1 задача), потом вливаем их в MASTER / MAIN.

какое отношение имеют ssh-ключи к GITHUB:
GitHub использует ключи SSH, чтобы можно было загружать файлы в репозиторий 
без ввода имени пользователя и пароля.

что позволяет делать сервис GitHub Pages.
Сервис Github Pages предоставляет бесплатный домен и хостинг, 
куда можно загружать файлы наших веб-сайтов, тестировать и вносить изменения.
Это публичный хостинг, т.е. любой желающий может получить доступ к исходнникам нашего сайта.


Приведите команды, которые выполняют след. действия (GIT):

создание, удаление веток;
создать: 	git branch bugfix
удалить: 	git branch -d bugfix
переключение между ветками: git checkout bugfix

загрузка и удаление веток на GitHub.

git push -u origin master
git push origin -delete bugfix


Коротко опишите, каким образом происходит публикация статического веб-сайта с помощью GitHub Pages:

создаём репозиторий на GitHub (create public repository)
переносим локальный репозиторий в репозиторий на GitHub:
git add .
git commit -m "Initial commit"
git remote add origin //сюда копируем линк из вкладки ssh: git@github...... (на GitHub)
git push -u origin master

последовательность наших шагов на GitHub: 
Open repository - Settings - Pages - Branch там выбираем master - Save
Через 2-10 мин после обновления страницы станет доступна 
ссылка на наш опубликованный сайт: your site is live at:...link...
