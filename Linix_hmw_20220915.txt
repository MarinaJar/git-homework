First level: Коротко дайте ответы на след. вопросы:

(1) Зачем предназначен файл .gitignore?
в нем указываются файлы, которые должны быть проигнорированы. 
Файлы .gitignore содержат шаблоны, которые сопоставляются с именами файлов
 в репозитории для определения необходимости их игнорировать.

(2) Что делает команда git remote rm origin?
Удаляет привязки старого (ранее удалённого) репозитория

(3) В каких случаях происходит MERGE CONFLICT?
конфликт слияния в GIT: 
Обычно конфликты возникают, когда два человека (или один) изменяют одни 
и те же строки в файле или один разработчик удаляет файл, 
который в это время изменяет другой разработчик. 
Git не может автоматически определить, какое изменение является правильным. 

(4) Как посмотреть отличия между ветками?
git diff master..bugfix
git diff bugfix master

(5) Какой командой удалить локальную и удаленную ветки?
git branch -D login (удаляет ветку независимо от её статуса)
git branch -d login

git push <remote_repository_name> --delete <branch_name> (удаление ветки, опубликованной в удалённом репозитории)
git push <remote_repository_name> :<branch_name>

(6) Как сделать файл неотслеживаемым (untracked)?
git rm --cached text.txt
